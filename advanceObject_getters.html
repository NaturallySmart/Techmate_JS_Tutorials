Advance Objects | Getters

In JavaScript, you can use getter methods to retrieve the value of an object's property when it is accessed through a specific reference or dot notation. A getter method is defined by using the "get" keyword followed by the desired property name and placing the code block in curly braces {}. The special "this" keyword inside a getter refers to the object itself.
Let’s take a look at a simple getter method: 

                let person = {
                    firstName: 'John',
                    lastName: 'Doe',
                    get fullName() {
                        return `${this.firstName} ${this.lastName}`;
                    }
                };
                // To call the getter method:
                console.log(person.fullName);  // Output: John Doe
Here's another example: 
                const person = {
                    _firstName: 'John',
                    _lastName: 'Doe',
                    getfullName() {
                        if (this._firstName && this._lastName) {
                            return `${this._firstName} ${this._lastName}`;
                            } else {
                                return 'Both firstName and lastName are required';
                            }
                        },
                }

                //To call the getter method:
                person.fullName; //Output: Both names

Output:

Notice that in the above getter method example above:

  • We use the get keyword followed by a function.
  • We use an if...else conditional to check if both _firstName and _lastName exist (by making sure they both return truthy values) and then return a different value depending on the result.
  • We can access the calling object’s internal properties using this. In fullName, we’re accessing both this._firstName and this._lastName.
  • In the last line we call fullName on person. In general, getter methods do not need to be called with a set of parentheses. Syntactically, it looks like we’re accessing a property.

Advantages of using Getter Method

  1. Encapsulation - Hides the underlying implementation from outside world.
  2. Control over data exposure - Allows you to control what parts of your code should have direct access to other parts.
  3. Maintainability - If you decide to change how you store or retrieve data later, only one place needs to be changed instead of multiple places.
  4. Getters can perform an action on the data when getting a property.
  5. Getters can return different values using conditionals.
  6. In a getter, we can access the properties of the calling object using this.
  7. The functionality of our code is easier for other developers to understand.
