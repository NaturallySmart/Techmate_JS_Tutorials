Advance Objects | Privacy

In this lesson we will learn about privacy in JavaScript objects and how to control access to object properties using getters and setters. We'll also discuss the concept of encapsulation which is a way of hiding internal implementation details within an object from outside code.

In JavaScript, accessing and updating object properties are fundametal. However, we sometimes do not want other developers to access or update certain object properties, that is, we do not want these object properties to be mutable or change in value.

In JavaScript, there is no built-in privacy feature, so as JavaScript developers, we need to follow a naming conventions that signal to other developers how to interact with a property. One common convention is to place an underscore _ before the name of a property to mean that the property should not be altered. Here’s an example of using _ to prepend a property. 

                const accountDetails = {
                    name: "John Doe",
                    _accountBalance: 50,000
                };
In the above example,
- The name property can be accessed by anyone who has access to the accountDetails object.
- The _accountBalance property is private because it starts with an underscore and it is not intended to be manipulated. However, it can be reassigned as 
accountDetails._accountBalance = 2000
